# ミツモリAI — LP デザイン指示書

> Engineer が Tailwind CSS で実装する際のビジュアルガイド

---

## 1. デザインコンセプト

**「信頼できる道具」** — 派手さではなく、堅実さ。工務店の社長が「これなら使えそうだ」と感じる実直なトーン。余白を十分にとり、情報を詰め込みすぎない。建設業の重厚感と、IT ツールの軽快さを両立させる。

---

## 2. カラーパレット

| 用途 | カラー | Tailwind | 意図 |
|---|---|---|---|
| メインカラー | `#1E3A5F` | `blue-900` 相当 | 紺色。信頼・堅実・プロフェッショナル |
| アクセントカラー | `#F59E0B` | `amber-500` | 琥珀色。CTA・強調。建設現場の安全色を連想 |
| アクセント（hover） | `#D97706` | `amber-600` | ボタン hover 時 |
| 背景（白） | `#FFFFFF` | `white` | ヒーロー・機能紹介・CTA セクション |
| 背景（薄グレー） | `#F8FAFC` | `slate-50` | 課題提起・FAQ セクション（交互配色） |
| 背景（紺） | `#1E3A5F` | — | 最終 CTA・フッター（白文字） |
| テキスト（本文） | `#1E293B` | `slate-800` | 可読性重視の濃いグレー |
| テキスト（補足） | `#64748B` | `slate-500` | サブコピー・注釈 |
| ボーダー | `#E2E8F0` | `slate-200` | テーブル・カード区切り |

### 配色ルール

- 背景は **白 → 薄グレー → 白 → 薄グレー → 紺** とセクションごとに交互に切り替え、視覚的にリズムを作る
- メインカラー（紺）は見出し・フッター・最終 CTA 背景に使用
- アクセント（琥珀）は CTA ボタンと「数字の強調」にのみ使用。乱用しない

## 3. タイポグラフィ

| 要素 | フォント | サイズ（PC） | サイズ（SP） | ウェイト |
|---|---|---|---|---|
| ヒーロー見出し | Noto Sans JP | 48px / `text-5xl` | 32px / `text-3xl` | Bold (700) |
| セクション見出し | Noto Sans JP | 32px / `text-3xl` | 24px / `text-2xl` | Bold (700) |
| サブ見出し | Noto Sans JP | 20px / `text-xl` | 18px / `text-lg` | SemiBold (600) |
| 本文 | Noto Sans JP | 16px / `text-base` | 16px / `text-base` | Regular (400) |
| 補足・注釈 | Noto Sans JP | 14px / `text-sm` | 14px / `text-sm` | Regular (400) |
| CTA ボタン | Noto Sans JP | 18px / `text-lg` | 16px / `text-base` | Bold (700) |

### フォント読み込み

```html
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@400;600;700&display=swap" rel="stylesheet">
```

### タイポグラフィルール

- 行間は `leading-relaxed`（1.625）を基本とする。本文の可読性を最優先
- 見出しは `leading-tight`（1.25）
- 英数字が混在する箇所（¥9,800 など）は `font-feature-settings: "palt"` で詰める

│  │  mb-6                                       │    │
│  │                                             │    │
│  │  [サブコピー]                                │    │
│  │  text-lg text-slate-500                     │    │

**ポイント:**
- ヒーローは画像なし。テキストのみで勝負する。工務店の社長は装飾より「何ができるか」を見る

- SP ではパディングを詰めて、ファーストビューに CTA まで収める

## 5. CTA ボタン仕様

全ページ共通の CTA ボタンスタイル:

```
bg-amber-500 hover:bg-amber-600
text-white font-bold
text-lg
px-8 py-4
rounded-lg
shadow-md hover:shadow-lg
transition-all duration-200
```

- テキスト: `7日間無料で試す →`
- `→` は `ml-2` でスペースを空ける
- hover で `translate-y-[-1px]` の微浮き上がりアニメーション
- ヒーローと最終 CTA の2箇所に配置。料金カード内にも1つ（計3箇所）

---

## 6. レスポンシブ方針

| ブレークポイント | 幅 | 対応 |
|---|---|---|
| SP | < 768px | 1カラム。パディング縮小。見出しサイズ縮小 |
| PC | >= 768px | max-width コンテナ。3ステップは3カラム |

- SP ファースト設計。工務店の社長はスマホで見る可能性が高い
- ハンバーガーメニューは不要（LP は単一ページ、ナビゲーションなし）
- コンテナ幅: `max-w-5xl mx-auto px-6`

---

## 7. アニメーション・インタラクション

最小限に抑える。過度な演出は工務店の社長には逆効果。

| 要素 | アニメーション |
|---|---|
| CTA ボタン | hover で微浮き上がり + shadow 拡大 |
| FAQ アコーディオン | `transition-all duration-200` で開閉 |
| セクション表示 | なし（スクロールアニメは不要） |

## 9. 実装時の注意

- **画像素材は使わない。** テキストとアイコン（Heroicons）のみで構成する。ストックフォトは信頼感を下げる
- **Heroicons（outline スタイル）** を統一的に使用する
- アクセシビリティ: ボタンは `focus:ring-2 focus:ring-amber-500 focus:ring-offset-2` を付与
- 表示速度: Web フォントは Noto Sans JP の 400/600/700 のみ。サブセット化推奨
- `<html lang="ja">` を忘れずに
