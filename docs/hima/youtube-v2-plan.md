## 1. 企画意図

### なぜこの動画を作るか

- X ローンチスレッドから流入したユーザーに、最短で「自分でも使える」を理解させる

- B4 記事（使い方ガイド）と連動し、オンボーディング離脱を減らす

- 視聴後 3 分以内に「レシピ作成 → CSV 読み込み → バッチ実行 → 結果エクスポート」の全体像を理解できる
- 「後で触る」ではなく「今すぐ触る」に転換させる

## 4. 構成（2〜3分）

### パート 1: フック（0:00 - 0:15）

- テキスト: 「50回のコピペを、1クリックに。」
- 実画面のハイライトを 2〜3 カットで先出し
- ナレーション: 「Hima の使い方を 3 分で見せます」

### パート 2: レシピ作成（0:15 - 0:55）

- レシピ名入力
- プロンプトテンプレートに `{{商品名}}` `{{カテゴリ}}` を記述
- 変数自動検出 UI を表示
- モデル選択（例: `gpt-4o`）
- 保存ボタン押下

### パート 3: CSV 読み込みとマッピング（0:55 - 1:25）

- CSV をドラッグ＆ドロップ
- カラムを変数へマッピング
- 入力件数表示を確認

### パート 4: バッチ実行（1:25 - 2:05）

- 「バッチ実行」押下
- 進捗バー、完了件数、行ステータスが更新される様子を表示
- 1 行でエラーケースも短く挿入（失敗行表示）

### パート 5: 結果確認とエクスポート（2:05 - 2:35）

- 結果テーブルで入力/出力対応を確認
- 行展開で出力全文を表示
- 「CSV エクスポート」押下

### パート 6: クロージング（2:35 - 2:55）

- 要点再掲: レシピ作成 → CSV → 実行 → エクスポート
- CTA: 「無料・登録不要。今すぐ試してください」
- 表示: `hima.agiinc.io` / `blog.agiinc.io`

## 6. ナレーション方針

- 説明は「機能紹介」ではなく「操作ガイド」にする
- 1 文を短くし、映像と発話の同期を優先
- 専門用語は必要最小限（BYOK は字幕注釈で補足）
- 速度はやや速め、間は短め（2〜3分に収める）
