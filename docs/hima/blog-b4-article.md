## Hima とは

**Hima（ヒマ）** は、AI への繰り返し作業を一括処理するバッチ AI ワークスペースだ。

プロンプトテンプレート（レシピ）を 1 つ書いて、CSV を放り込むだけ。10 件でも 100 件でも 1,000 件でも、AI がまとめて処理する。結果はテーブルで確認し、CSV でエクスポート。

50 回のコピペを、1 クリックに。

## クイックスタート — 3 ステップで完了

### Step 1: API キーを登録する

画面右上の **設定（⚙）** を開き、使いたい AI プロバイダの API キーを入力する。

対応プロバイダ:

- **OpenAI** — gpt-4o, gpt-4o-mini
- **Anthropic** — claude-sonnet-4-5, claude-haiku-4-5
- **Google AI** — gemini-2.0-flash

キーは **ブラウザの localStorage に保存** される。サーバーには一切送信しない。登録後に接続テストができるので、キーが有効か確認してほしい。

### Step 2: レシピを書く

レシピ = プロンプトテンプレートだ。左パネルのエディタに、プロンプトを書く。

普通のプロンプトとの違いは `{{変数}}` が使えること。二重波括弧で囲んだ部分が、CSV のカラムに対応する変数になる。

たとえば、こう書く:

```
あなたはプロのコピーライターです。
以下の商品の魅力的な説明文を150字以内で書いてください。

商品名: {{product_name}}
カテゴリ: {{category}}
特徴: {{features}}
```

エディタが `{{product_name}}` `{{category}}` `{{features}}` を自動で検出し、変数リストに表示する。

レシピには名前を付けて保存できる。一度作ったレシピは何度でも再利用可能だ。

### Step 3: CSV を読み込んで実行

右パネルで CSV ファイルをドラッグ＆ドロップ（またはテキスト貼り付け）する。

CSV のカラムとレシピの変数を **マッピング** する。`{{product_name}}` → CSV の A列、`{{category}}` → B列、のように紐づけるだけだ。

準備ができたら:

1. **テスト実行** — まず 1 件だけ試す。出力を確認してレシピを調整
2. **バッチ実行** — 全件を一括処理。プログレスバーで進捗がわかる
3. **CSV エクスポート** — 結果をダウンロード。入力カラムと出力カラムの両方が含まれる

処理は **最大 5 件が並列** で動く。100 件なら数分で完了する。

## レシピの書き方 — 実例 3 つ

### 変数の基本ルール

- `{{変数名}}` の形式で記述する。英数字とアンダースコアが使える
- 変数はリアルタイムで自動検出される
- 同じ変数名をプロンプト内で複数回使える
- CSV のカラム名と変数名は一致しなくていい（マッピング UI で紐づける）

### 実例 1: 商品説明の生成

**レシピ:**

```
以下の商品について、ECサイト向けの説明文を200字以内で書いてください。
ターゲット顧客は{{target}}です。

商品名: {{product_name}}
価格帯: {{price_range}}
主な特徴: {{features}}

トーン: カジュアルで親しみやすく
```

**CSV の例:**

| product_name | price_range | features | target |
|---|---|---|---|
| ワイヤレスイヤホン X1 | 5,000〜8,000円 | ノイキャン、防水、12時間連続再生 | 通勤する20代 |
| ポータブル充電器 Z3 | 3,000〜5,000円 | 10,000mAh、急速充電、軽量 | 外出が多い社会人 |

### 実例 2: メール文面の作成

**レシピ:**

```
以下の情報をもとに、ビジネスメールの文面を作成してください。

送信先: {{recipient}}
用件: {{subject}}
伝えたいポイント: {{key_points}}
トーン: {{tone}}

署名は不要です。本文のみ出力してください。
```

1 つのレシピで、取引先への連絡も社内通知も、トーンを変えるだけで対応できる。

### 実例 3: データの分類

**レシピ:**

```
以下のカスタマーレビューを読み、感情を分類してください。

レビュー: {{review_text}}

出力フォーマット:
感情: [ポジティブ / ネガティブ / ニュートラル]
理由: (一行で)
```

数百件のレビューも一括分類。結果を CSV でエクスポートすれば、スプレッドシートで集計できる。

## FAQ

### BYOK とは？

**Bring Your Own Key** の略。Hima は AI の API を独自に提供しない。あなた自身の OpenAI / Anthropic / Google AI の API キーを持ち込んで使う方式だ。

**メリット:**

- Hima の利用自体は完全無料。API の従量課金のみ
- モデルの選択肢が広い。用途に応じて最適なモデルを使い分けられる
- ベンダーロックインがない。いつでも別のモデルに切り替えられる

**API キーの取得方法:**

- OpenAI: [platform.openai.com](https://platform.openai.com) でアカウント作成後、API keys ページで発行
- Anthropic: [console.anthropic.com](https://console.anthropic.com) でアカウント作成後、API keys ページで発行
- Google AI: [aistudio.google.com](https://aistudio.google.com) で API キーを発行

### 対応モデル一覧

| プロバイダ | モデル | 特徴 |
|---|---|---|
| OpenAI | gpt-4o | 高品質な汎用モデル。複雑なタスクに |
| OpenAI | gpt-4o-mini | 軽量・高速・低コスト。大量処理向き |
| Anthropic | claude-sonnet-4-5 | 高精度。長文の生成や分析に強い |
| Anthropic | claude-haiku-4-5 | 高速・低コスト。シンプルなタスクに |
| Google AI | gemini-2.0-flash | 高速。コスト効率が良い |

迷ったら **gpt-4o-mini** か **claude-haiku-4-5** から始めるのがおすすめだ。速くて安い。品質を上げたいときに上位モデルに切り替えればいい。

### データの安全性

Hima は **完全クライアントサイド** で動作する。

- API キーは **ブラウザの localStorage** に保存される。サーバーには送信しない
- レシピも localStorage に保存される
- バッチの入出力データは **メモリ上にのみ存在** する。タブを閉じると消える。必要な結果は CSV エクスポートで保存してほしい
- AI プロバイダへのリクエストは、ブラウザから直接ではなく **Cloudflare Worker 経由** で送信される。これは AI プロバイダの API がブラウザからの直接アクセス（CORS）を許可していないためだ。Worker はリクエストをそのまま転送し、CORS ヘッダーを付与するだけで、データの保存やログ記録は一切行わない

### 処理中にタブを閉じたら？

バッチ結果はメモリ上にしか存在しない。タブを閉じると結果は失われる。処理が終わったら忘れずに CSV エクスポートしてほしい。

### 料金は？

Hima 自体は **完全無料** だ。かかるのは AI プロバイダの API 利用料のみ。各プロバイダの料金体系に従う。
