## 1. ブログ B4 使い方ガイド — APPROVE

**想定文字数: 約 3,500 字（目標 3,000〜4,000 字の範囲内）**

### 良い点

- 記事構成が launch-content-strategy.md セクション 2-B の B4 企画案と完全に整合。導入 → クイックスタート → レシピの書き方 → ユースケース → FAQ → AGI Inc. について、の 6 セクション構成
- 「50 回のコピペを、1 クリックに。」のワンライナーが記事前半で刺さる。タイムラインで共有されたときの引用文になり得る
- 3 ステップの説明が明瞭。技術者以外にも伝わる粒度
- レシピの実例 3 つ（商品説明 / メール文面 / データ分類）がターゲット（AI 好きのプロシューマー）の実務に直結
- `{{変数}}` の説明が具体的。CSV サンプルのテーブル表示でイメージが湧く
- FAQ セクションの BYOK 説明が丁寧。API キー取得先の URL 一覧は初心者に親切
- CORS プロキシの存在を隠さず説明している点が技術者の信頼を獲得する。「Worker はリクエストをそのまま転送し、CORS ヘッダーを付与するだけで、データの保存やログ記録は一切行わない」の一文が重要
- 対応モデル一覧テーブルの「迷ったら gpt-4o-mini か claude-haiku-4-5 から」の推薦は初心者の意思決定コストを下げる。的確
- 末尾の CTA がシンプル。URL 1 つ + X ハンドル 1 つ。過不足なし
- X ハンドル @agilab_agiinc は AMENDMENTS.md #001 および全投稿と一貫性あり

### 軽微な改善案（任意）

- フロントマター（`emoji`, `type`, `topics`）は B1 記事と同一形式で統一されている。blog.agiinc.io のデプロイ時にプラットフォームが受け付けるか確認すること。問題があれば公開前に調整
- 「処理は最大 5 件が並列で動く。100 件なら数分で完了する。」— プロダクトの実装と一致していることを CTO に確認推奨。launch-content-strategy.md の「CSV 100 件を 3 分で処理」と整合はしている
- B1 記事へのリンクが `https://blog.agiinc.io` のルートになっている。公開時に正確なスラッグ付き URL に差し替えること
