### 3-1. A1: デモGIF 1-B（X投稿用）

#### 基本仕様

| 項目 | 仕様 |
|---|---|
| 用途 | ローンチスレッドT5 + D2（3/19 LP公開投稿） |

#### シーン構成（6シーン）

| # | 秒数 | 内容 | 映像ソース | 演出ノート |
|---|---|---|---|---|
| S0 | 0:00-0:02 | ワークスペース全景 | `assets/real-ui/s0-overview.webm` | 静止ショット。カーソル中央 |
| S1 | 0:02-0:08 | レシピ作成（変数入力+自動検出） | `assets/real-ui/s1-recipe.webm` | 実UIでのタイプ操作を直接録画 |
| S2 | 0:08-0:14 | CSVドラッグ&ドロップ+マッピング | `assets/real-ui/s2-csv-import.webm` | ドロップゾーンのボーダー色変化を含む |
| S3 | 0:14-0:22 | バッチ実行（進捗バー→完了） | `assets/real-ui/s3-batch-exec.webm` | Tealの進捗バーがダーク背景で高コントラスト |
| S4 | 0:22-0:28 | 結果テーブル+CSVエクスポート | `assets/real-ui/s4-export.webm` | テーブル行が読める程度のフォントサイズ（最小12px） |
| End | 0:28-0:30 | エンドフレーム | Remotion生成 | 背景`#1A1A1A`、白テキスト、2秒間表示 |

#### ダークテーマ適応（`dark-theme-adaptation-plan.md` 準拠）

Session 19で確定したダークテーマ適応方針に基づく:

| 項目 | 対応内容 |
|---|---|
| **方針** | 実UI映像のフル置換。モック+実UIのハイブリッドは不採用 |
| **UIベース背景色** | ダークネイビー `#0F172A`（実UIの実際の色） |
| **トランジション背景** | シーンコンテナ `backgroundColor: #0F172A`。白フラッシュ防止 |
| **カーソル** | Kap録画に含まれるカーソルを使用。Remotion側のカーソルオーバーレイ無効 |
| **エンドフレーム接続** | ダークネイビー→チャコール`#1A1A1A`のフェードは自然。変更不要 |
| **GIF圧縮** | ダークテーマはライトテーマより圧縮効率が高い（暗色領域のフレーム間差分が少ない） |

#### エンドフレーム仕様

| 項目 | 仕様 |
|---|---|
| 背景 | チャコール `#1A1A1A` |
| メインテキスト | `hima.agiinc.io`（Inter Bold 24px, 白 `#FFFFFF`、中央配置） |
| サブテキスト | `無料・登録不要・BYOK`（Inter Regular 14px, 白 `#FFFFFF`、URL下8px） |
| 表示時間 | 2秒 |
| フェード | S4→End: `fade()` 0.5s |

#### キャプチャ環境

| 項目 | 設定 |
|---|---|
| ブラウザ | Chrome（タブバー・アドレスバー非表示） |
| ウィンドウサイズ | 1440 x 900px |
| 録画ツール | Kap → WebM出力 |
| ダミーデータ | 「商品説明文生成」ユースケース（`docs/hima/demo-data.csv`、10件） |
| マウスカーソル | 滑らかな動き。Kapネイティブのクリックハイライトを使用 |
| ディスプレイ輝度 | 80%以上推奨 |

#### ffmpeg変換パラメータ

```bash
# パレット生成（ダークテーマ最適化）
ffmpeg -i input.webm -vf "fps=12,scale=1200:-1:flags=lanczos,palettegen=max_colors=256:stats_mode=diff" palette.png

# GIF変換
ffmpeg -i input.webm -i palette.png -lavfi "fps=12,scale=1200:-1:flags=lanczos[x];[x][1:v]paletteuse=dither=floyd_steinberg" -loop 0 demo-1b.gif
```
