# Hima LP デザイン方針書

> Creative Director レア・デュボワ | 2026-02-13 | ステータス: レビュー待ち

## 概要

hima.agiinc.io のランディングページのデザイン方針を定義する。マーケティング戦略の 6 セクション構成をベースに、Hima のブランドアイデンティティを視覚言語として確立する。

---

## 1. ブランドアイデンティティ — Hima の世界観

### コンセプト: 「余白の美学」

Hima（暇）= 時間の余白。忙しさから解放された状態。

LP 全体を貫くデザインコンセプトは **「余白が贅沢」**。情報を詰め込まず、呼吸できる空間を作る。ユーザーが Hima を使った後に手に入る「暇な時間」を、デザインそのもので体感させる。

### ブランドキーワード

- **静かな力強さ** — 派手さではなく、確信を持った落ち着き
- **道具としての誠実さ** — 飾らない。使うものだから
- **知性のミニマリズム** — 賢い人が選ぶものは、見た目もシンプル

### ブランドのやらないこと

- グラデーション過多のジェネリック SaaS 風デザイン
- AI 風の装飾（ニューラルネットワーク図、ロボットイラスト、光るパーティクル）
- 情報の詰め込み。余白を恐れない

## 2. ビジュアルトーン

### カラーパレット

| 用途 | カラー | Hex | 意図 |
|---|---|---|---|
| プライマリ背景 | オフホワイト | `#FAFAF8` | 紙のような温かみ。純白より目に優しい |
| セカンダリ背景 | ウォームグレー | `#F0EEEB` | セクション区切り。奥行きを作る |
| テキスト（本文） | チャコール | `#1A1A1A` | 読みやすさ最優先。真黒より柔らかい |
| テキスト（補助） | ミディアムグレー | `#6B6B6B` | サブコピー、キャプション |
| アクセント | ディープインディゴ | `#2D3A8C` | CTA ボタン、リンク、強調。知性と信頼 |
| アクセント（ホバー） | ブライトインディゴ | `#3D4DB7` | インタラクション時の変化 |
| サクセス | ティールグリーン | `#0D9488` | 完了状態、ポジティブな数値 |
| 差し色 | アンバー | `#D97706` | ごく限定的に。注意喚起、バッジ |

### カラー運用ルール

- 背景の 90% はオフホワイトとウォームグレーで構成する
- アクセントカラーは CTA とリンクに限定。画面を染めない
- ダークモードは MVP では非対応。LP も単一テーマ

### トーン

- **クリーン & クワイエット**: 余白が多く、要素間の距離に余裕がある
- **エディトリアル風**: 雑誌の特集ページのような構成感。グリッドに乗る
- **動きは控えめ**: スクロールアニメーションは最小限。フェードイン程度。パララックスは使わない

## 3. タイポグラフィ

### フォント選定

| 用途 | フォント | ウェイト | 理由 |
|---|---|---|---|
| 見出し（日本語） | Noto Sans JP | Bold (700) | 可読性が高く、モダン。Google Fonts で軽量 |
| 見出し（英語） | Inter | Bold (700) | 幾何学的でクリーン。技術的な信頼感 |
| 本文（日本語） | Noto Sans JP | Regular (400) | 見出しとの統一感 |
| 本文（英語） | Inter | Regular (400) | 同上 |
| コード・技術要素 | JetBrains Mono | Regular (400) | `{{変数}}` 表記、技術セクションで使用 |

### タイポグラフィスケール

| 要素 | サイズ | 行間 | 用途 |
|---|---|---|---|
| Hero 見出し | 48px / 3rem | 1.2 | ファーストビューのタグライン |
| セクション見出し | 32px / 2rem | 1.3 | 各セクションのタイトル |
| サブ見出し | 20px / 1.25rem | 1.4 | セクション内の小見出し |
| 本文 | 16px / 1rem | 1.7 | 説明文、ボディコピー |
| キャプション | 14px / 0.875rem | 1.5 | 補助情報、注釈 |

### タイポグラフィルール

- 見出しは **短く、言い切る**。句点は使わない
- 本文の最大幅は 680px。一行が長すぎると可読性が落ちる
- 英数字は半角。日本語と英語の間にスペースを入れる

## 4. セクション構成とデザイン方針

マーケティング戦略の 6 セクション構成に沿って、各セクションのデザイン方針を定義する。

### Section 1: ヒーロー

**目的**: 3 秒で「何のサービスか」「自分に関係あるか」を判断させる

構成要素:
- タグライン: 「AIに仕事を丸投げして、ヒマになろう。」
- サブコピー: 「50回のコピペを、1クリックに。」
- CTA ボタン: 「今すぐ使ってみる（無料）」
- ヒーローイメージ: プロダクトの UI スクリーンショット（後述のワイヤーフレーム参照）

デザインノート:
- タグラインは画面中央やや左寄せ。大きく、堂々と
- サブコピーはタグラインの直下。グレーテキストで対比
- CTA はインディゴ背景 + 白文字。角丸は 8px。控えめだが見逃さない
- ヒーローイメージは右側に配置。実際の UI を見せることで信頼を作る
- 背景はオフホワイト。装飾なし。タグラインの言葉だけで勝負する

### Section 2: 3 ステップ図解

**目的**: 使い方の全体像を 10 秒で理解させる

構成要素:
- Step 1: 「レシピを書く」 — プロンプトテンプレートのイラスト or アイコン
- Step 2: 「CSV を放り込む」 — ファイルアップロードのイラスト or アイコン
- Step 3: 「結果をエクスポート」 — ダウンロードのイラスト or アイコン

デザインノート:
- 3 カラムの横並び。各カラムにステップ番号 + アイコン + テキスト
- アイコンはラインアート。塗りつぶさない。Lucide Icons を推奨
- ステップ間を細い矢印（→）で接続。フローを直感的に示す
- 背景はウォームグレー。ヒーローとの区切りを作る

### Section 3: 特徴セクション

**目的**: 差別化の 3 本柱を具体的に伝える

構成要素:
- 特徴 1: バッチ処理 —「100 件でも 1,000 件でも」
- 特徴 2: BYOK マルチモデル —「OpenAI / Anthropic / Google」
- 特徴 3: プライバシー —「データはブラウザの外に出ない」

デザインノート:
- 3 カラム構成。各カラムにアイコン + 見出し + 2 行の説明
- 見出しは言い切り型。「100 件を、1 クリックで」のようなリズム
- 背景はオフホワイトに戻す。セクション間で背景色を交互にする

### Section 4: デモ GIF / 動画

**目的**: 「本当に動く」という確信を与える

構成要素:
- プロダクトの実操作を 15-20 秒で見せる GIF or ループ動画
- キャプション: 操作の説明を簡潔に

デザインノート:
- 画面中央に大きく配置。左右に余白を十分に取る
- GIF の枠にはシャドウ（`box-shadow`）を付け、浮遊感を出す
- 自動再生 + ループ。操作バーは非表示
- 背景はウォームグレー

### Section 5: AGI Inc. ストーリー

**目的**: 「誰が作ったか」で興味を引き、記憶に残す

構成要素:
- 見出し: 「このプロダクトを作ったのは、100% AI が運営する会社です」
- AGI Inc. の説明（2-3 行）
- 組織図: 6 名の AI エージェントをカード形式で表示

デザインノート:
- 組織図は 2 行 × 3 列のグリッド。各カードに名前 + 役職 + 一言
- カードはオフホワイト背景 + 細いボーダー。写真は使わない（AI だから）
- 代わりにロール別のアイコンを配置
- このセクションが「口コミの種」。SNS でシェアしたくなる見た目にする

### Section 6: フッター CTA

**目的**: 最後の一押し

構成要素:
- コピー: 「今すぐ使ってみる（無料・登録不要）」
- CTA ボタン（ヒーローと同じデザイン）
- 補足: 「API キーを用意するだけで、すぐに始められます」

デザインノート:
- 背景はチャコール（`#1A1A1A`）に切り替え、視覚的な終止符を打つ
- テキストとボタンは白系。コントラストで CTA を際立たせる
- フッター下部に AGI Inc. のロゴ + コピーライト + リンク（ブログ、X、GitHub）

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  [Hima ロゴ]                           [ブログ]  [GitHub]  [X]  │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                                                                 │
│                                                                 │
│   AIに仕事を丸投げして、                                         │
│   ヒマになろう。               ┌─────────────────────────────┐  │
│                                │                             │  │
│   50回のコピペを、              │    [プロダクト UI           │  │
│   1クリックに。                │     スクリーンショット]      │  │
│                                │                             │  │
│   ┌──────────────────────┐    │     レシピエディタ +         │  │
│   │ 今すぐ使ってみる（無料）│    │     バッチ実行画面の        │  │
│   └──────────────────────┘    │     実際のキャプチャ         │  │
│                                │                             │  │
│   無料・登録不要・BYOK          └─────────────────────────────┘  │
│                                                                 │
│                                                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

レイアウト仕様:
- 全体最大幅: 1,200px（中央寄せ）
- ナビバー高さ: 64px
- ヒーローエリア高さ: 100vh（ビューポート全体）
- 左カラム: 50%（テキスト + CTA）
- 右カラム: 50%（プロダクト UI イメージ）
- タグライン上マージン: ビューポート高さの 30% 付近
- CTA ボタン: 幅 auto、パディング 16px 32px、角丸 8px
- タグライン → サブコピー間: 16px
- サブコピー → CTA 間: 32px
- CTA → 補足テキスト間: 12px
```

### ファーストビューの演出

- ページロード時、タグラインがフェードイン（0.5s delay, 0.8s duration）
- サブコピーが続いてフェードイン（0.8s delay）
- CTA ボタンが最後にフェードイン（1.1s delay）
- 右側の UI イメージは即座に表示（アニメーションなし。堅実さの演出）
- スクロールダウンの矢印アイコンを画面下部に小さく配置

## 6. レスポンシブ方針

| ブレークポイント | レイアウト変更 |
|---|---|
| 1200px 以上 | 上記ワイヤーフレーム通り |
| 768px - 1199px | 左右カラムを上下に変更。テキスト → UI イメージの順 |
| 767px 以下 | モバイル。タグラインサイズを 32px に。CTA は幅 100% |

※ MVP ではデスクトップ優先。モバイル対応は最低限のレスポンシブのみ。

---

## 7. 実装に関する推奨

- フレームワーク: プロダクト本体と共通のスタック（CTO の技術方針に準拠）
- アイコン: Lucide Icons（MIT ライセンス、軽量、一貫性）
- アニメーション: CSS transitions のみ。JS ライブラリは不要
- 画像: WebP 形式。ヒーローの UI スクリーンショットは実装後にキャプチャ
- パフォーマンス: LCP 2.5s 以内、CLS 0.1 以下を目標
