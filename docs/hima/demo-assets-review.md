## 1. マーケティング効果

### 評価: A

**デモ GIF（1-A, 1-B）**
- 4 シーン構成（レシピ → CSV → バッチ → エクスポート）はユーザージャーニーそのもの。AI 好きプロシューマーは「動いている実画面」を最も信頼する。モックアップ不可の方針も正しい
- X 用 GIF に S0（ワークスペース全景）を追加したのは良判断。X のタイムラインでは冒頭 2 秒で「何のツールか」を伝える必要がある
- エンドフレームに `hima.agiinc.io` + 「無料・登録不要・BYOK」を入れたのは CTA として機能する

**Before/After 画像（03/28）**
- 「60 min → 3 min」の対比は数字好きのプロシューマーに刺さる。ミッション「人間を暇にする」の視覚的証明として最強クラスのアセット

**カウントダウン / ティーザー**
- シンプルで潔い。情報の詰め込みを避けている点が LP デザイン方針の「余白の美学」に合致

**所見:**
- デモ GIF のタイプライター演出（1 文字ずつ表示）は、操作感のリアリティを出す良いアイデア。ただし速度が遅すぎると退屈になる。1 文字あたり 50-80ms を推奨
- 数値ダッシュボードテンプレの 4 指標（UV / バッチ実行 / 処理行数 / 使用モデル TOP）は適切。ローンチ初日のストーリーとして十分な粒度

## 2. ブランド一貫性

### 評価: A

LP デザイン方針書との整合性は高い。

| チェック項目 | 結果 |
|---|---|
| カラーパレット準拠（オフホワイト / チャコール / ディープインディゴ） | OK |
| フォント準拠（Noto Sans JP / Inter / JetBrains Mono） | OK |
| トーン準拠（クリーン、余白多い、AI 装飾なし） | OK |
| 画像形式準拠（WebP / GIF / WebM） | OK |
| box-shadow 値の一致（LP Section 4 と 1-A の値が同じ） | OK |

| Lucide Icons 使用（MIT ライセンス、LP 推奨と一致） | OK |

**問題点 1 件（要修正）:**

> **BYOK 訴求画像（03/25 ローンチスレッド用）に OpenAI / Anthropic / Google のロゴアイコンを使用する指定がある。** 各社のブランドガイドラインでは、ロゴの使用に制約がある場合が多い。特に「第三者プロダクトの販促素材」での使用は許諾が必要なケースがある。

**改善提案:** ロゴアイコンの代わりに、テキストで「OpenAI / Anthropic / Google」を Inter Bold で横並びに配置する案を推奨。または Lucide の汎用アイコン（`brain`, `cpu`, `sparkles` など）で 3 つのモデルプロバイダーを抽象的に表現する。ブランドリスクを回避しつつ訴求力を保てる。

## 3. スケジュールの現実性

### 評価: B+

全体のタイムラインは堅実。ただし 2 点のバッファ不足と 1 点の依存リスクがある。

**問題点 2 件（要修正）:**

> **納品日 = 使用日の違反が 2 件ある。** 仕様書冒頭で「使用日の 2 営業日前までに完成」と定義しているが、以下が違反している:

| # | アセット | 納品日 | 使用日 | バッファ |
|---|---|---|---|---|
| 11 | ユースケース Before/After | 03/28 | 03/28 | **0 日（違反）** |
| 12 | 週間サマリー画像 | 03/30 | 03/30 | **0 日（違反）** |

**改善提案:**
- #11 Before/After: Before 側（ChatGPT イメージ図）は事前制作可能。03/26 納品に前倒しし、After 側の Hima スクリーンショットのみローンチ後に差し込む 2 段階方式にする
- #12 週間サマリー: テンプレートを 03/24 に事前制作し、実績数値のみ 03/30 当日に差し込む方式にする（数値ダッシュボードと同じアプローチ）

**クリティカルパスのリスク評価:**

```
リスク高 ████████░░ UI 実装完了 → 画面キャプチャ（03/12）
リスク中 ██████░░░░ YouTube V2 制作（03/18-20、3 日間で TTS + 動画編集）
リスク低 ████░░░░░░ OGP 画像（依存少、早期着手可能）
```

- **最大リスク: UI 実装完了日の未確定。** 03/12 のキャプチャ日に UI が間に合わなければ、デモ GIF・ヒーロー画像・YouTube 素材の全てが連鎖遅延する。CTO / Engineer から 03/12 の確約を取る必要がある。リスク軽減策（03/14 延期 + 土日制作）は現実的だが、土日制作はリスク軽減策として常態化させたくない
- **YouTube V2 制作が 3 日間は攻めている。** Remotion 編集 + TTS 生成 + レンダリングを 03/18-20 で完了する想定。TTS の品質調整（イントネーション、間）に想定外の時間がかかるリスクがある。リスク軽減策として台本の TTS 向けリライト（読みやすさ最適化）を 03/17 以前に完了させておくことを推奨
