## I-01〜I-04 正式検証（3/22）詳細

### 3/22 共通実施ルール

- 実施前: 各サービスのデプロイコミットと監視対象 URL の最終確認
- 記録: 手順、HTTP ステータス、主要画面キャプチャ、想定外エラーを同日メモに残す
- 判定: 合否はいずれも Go 条件を満たした場合のみ Green、要修正は Yellow、停止は Red とする

### I-01 `hima.agiinc.io` 稼働確認

- CTO、Engineer 共同
- チェック:
  - HTTPS / HSTS / メイン導線アクセス（`/`、`/app`）の正常表示
  - API キー未設定時ガードが想定どおり表示されること
  - バッチ処理 UI を 1 回起動し、終了状態が適切にレンダリングされること
- 合格基準:
  - 主要画面が 200 で表示
  - 重要操作で 500 系エラーゼロ

### I-02 `api-hima.agiinc.io` 稼働確認

- CTO、Engineer 共同
- チェック:
  - 健全性エンドポイント応答（GET `/` 等）
  - OPTIONS / 事前許可 Origin の CORS 応答
  - 非許可 Origin の CORS 非付与
  - 3 プロバイダ中継経路の疎通
- 合格基準:
  - Origin 制御が前提どおりで、重要エンドポイントに 2xx が確認できること

### I-03 `agiinc.io` 稼働確認

- CMO、CTO 共同
- チェック:
  - トップ表示と全主要導線リンクの有効性
  - 会社サイトから `hima.agiinc.io` / `blog.agiinc.io` へのリンク遷移
  - 重要メタ情報（タイトル、description、リンク先）が表示されること
- 合格基準:
  - トップ表示が安定、導線遷移が失敗しないこと

### I-04 `blog.agiinc.io` 稼働確認

- CMO、CTO 共同
- チェック:
  - B1/B4 記事の本文表示
  - OGP / 画像表示
  - サイト内リンクの有効性（記事間・会社サイト導線）
- 合格基準:
  - 記事ページの 404 ゼロ
  - 主要リンク導線が維持されること
